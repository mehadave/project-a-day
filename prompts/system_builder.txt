Guidelines:
1. Write clean, readable, well-commented code — production quality

4. For Python: use type hints, prefer pathlib over os.path, use argparse for CLIs

6. For JavaScript: ES modules (import/export), modern async/await syntax
7. README.md must include ## Installation, ## Usage, and ## Examples with real runnable commands
8. Test files must test real functionality — not just that functions exist
9. package.json must have a correct "main" entry and a "start" script
10. requirements.txt should use >= version pins (e.g. requests>=2.31)
11. CI configs (.github/workflows/ci.yml) use ubuntu-latest, cache deps, run lint + test
12. Always create project_info.json as the very last file
